# 基础设施建设与开发环境

基础设施建设极其重要，如果没有良好的基础设施建设，开发与调试就会进入到想一出是一出的境地，因此基础设施的建设是极其重要的。

而开发环境实际上也是基础设施的一部分，只是它过于重要，因此我们会单独把它拿出来讲。

## 什么是基础设施建设？

很简单一个道理，如果你之后可预见有一个必然会遇到的问题，那么你就应该提前做好准备，做好准备的过程就是基础设施建设，可复用的准备就是你的基础设施。

由于我们进行二维码扫描的开发一定会需要二维码，那么我们应该找出如何生成符合条件的二维码，是调试时即时生成还是各个情况生成之后打印成纸质备用；由于我们视觉开发需要有视觉图像，那么我们应该准备如何获取到开发阶段的视觉图像，是做个仿真模型还是实物，是在摄像头装到车上录像还是直接用手机录像。我们需不需要打印物料？需不需要准备地图？这不仅是视觉需要考虑的，也是整个团队在预研过程中需要考虑清楚的。

## 开发环境的建设

前面我们提到，我们将会使用嵌入式Linux设备作为我们视觉的算力平台，但是我们真的要一开始就在这样一个性能孱弱的嵌入式Linux设备上面搞开发吗？答案是否定的。

如果你使用传统视觉开发算法，你没有依赖问题，你完全可以在PC上开发。甚至于如果你是一个纯Python的程序，你直接使用Windows也没有问题。

如果你使用深度学习算法，那么会有依赖问题，主要是算力设备在PC上与嵌入式板卡上不同导致的，但也不意味着你在原型阶段就要上板。你完全可以在PC上进行开发，对于推理部分使用ONNX这种兼容性好的格式，等到框架流程都走通，再移植到具体的算力平台与板卡上。

如果使用 C++，则建议一开始就在 Linux 环境下开发。因为 Windows 和 Linux 的 C/C++ 运行时库实现不同，你在 Windows 上调通的代码可能会因为用了 POSIX 专有 API 或 Linux 特有的系统调用而无法在嵌入式 Linux 上直接编译。更别说 GPIO、I2C、SPI 这类硬件接口了，Windows 上根本没有——直接从 Linux 环境起步可以避免后期移植时踩坑。

## Linux 环境的选择

常见的Linux环境选择主要有这样几种：

虚拟机Linux、真机/双系统Linux、WSL Linux、Docker。

推荐使用WSL，也就是Windows Subsystem for Linux。WSL是微软提供的一个在Windows上运行Linux环境的工具，使用WSL可以让你在Windows上直接运行Linux命令行工具和应用程序，而不需要安装虚拟机或双系统。

如果有兴趣也可以使用物理机（真机）Linux。

Linux环境的使用，大部分操作现在已经可以在不断查找AI或者直接使用Coding Agent来完成了，但我依然建议针对Linux的常用命令与逻辑进行基本的学习。推荐资料是[Linux101教程](https://101.lug.ustc.edu.cn/)。

关于WSL环境具体的搭建，推荐微软官方文档，你也可以让AI解决，现在WSL案例已经是一行命令就解决，不用再自己进行使用各种设置了。如果你查了文档发现要这样做，建议尽早放弃使用CSDN。

## 我们还需要什么基础设施

- 上一节讲到我们将会使用神经网络，因此当然就会需要一个数据标注平台来对你的数据进行标注。目前比较常见的开源数据标注平台有LabelImg、LabelMe、LabelStudio等，推荐使用LabelStudio，功能比较全，支持多种标注类型，界面也比较友好。
- 版本控制工具。无论是个人开发还是团队开发，版本控制工具都是必不可少的。推荐使用Git，配合GitHub、Codeberg等平台进行代码托管和协作。
- 板卡连接工具。对于嵌入式Linux开发来说，常见的连接工具有SSH、串口终端（如PuTTY、minicom）等。建议熟悉这些工具的使用，以便于调试和部署。

## 总结

基础设施建设是一个重要的环节，好的基础设施可以大大提高开发效率和调试效率。我们需要根据项目的需求和团队的情况来选择合适的基础设施，并且在开发过程中不断完善和优化这些基础设施，以支持我们的开发工作。
